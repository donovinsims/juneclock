# Juneclock / Clockout

TanStack Start marketing site for Clockout (done-for-you automation for local service businesses).

## Stack

- **React 19** + **TanStack React Router v1** (file-based) + **TanStack Start** (SSR)
- **Tailwind CSS v4** — no `tailwind.config.js`; CSS-driven config via `@theme inline` in `src/styles.css`
- **shadcn/ui** (new-york style, Radix primitives, lucide-react icons)
- **Bun** runtime & package manager
- **Vite 7** with Nitro SSR (Vercel preset via `nitro/vite` plugin)
- **TypeScript 5.8** strict, `noUnusedLocals`/`noUnusedParameters` OFF, `skipLibCheck` ON
- **React Query** + **Zod** + **react-hook-form** + **motion** (framer-motion successor)

## Commands

```sh
bun run dev        # Vite dev server
bun run build      # production build
bun run build:dev  # dev-mode build
bun run preview    # preview production build
bun run lint       # ESLint 9 flat config
bun run format     # Prettier (100-width, double quotes, trailing commas)
```

No test framework is installed.

## Routes (file-based)

All routes live in `src/routes/`. Do **not** create `src/pages/` or `app/` — those are other frameworks.

| File | URL pattern |
|---|---|
| `__root.tsx` | App shell (no layout nesting — `<Outlet />` at root) |
| `index.tsx` | `/` |
| `services.tsx` | `/services` (parent — child routes need special handling) |
| `services.$slug.tsx` | `/services/:slug` (param via `$slug`, not `:slug`) |
| `*.tsx` | Flat name matches URL |

`routeTree.gen.ts` is auto-generated by TanStack Router — **never edit by hand**.

## Architecture

```
src/
  lib/utils.ts          # cn() — clsx + tailwind-merge
  lib/config.server.ts  # server-only env config (.server.ts suffix prevents client bundling)
  lib/error-capture.ts  # SSR error capture
  lib/error-page.ts     # SSR error page renderer
  lib/api/              # server function examples
  components/
    site/               # page-specific components (Hero, Header, Footer, etc.)
    ui/                 # shadcn components (46 files — treat as library, add new ones via shadcn CLI)
  data/                 # static content (faqs, industries, offer, serviceArea)
  router.tsx            # Router creation (used internally — do not import to edit routes)
  start.ts              # TanStack Start instance with error middleware
  server.ts             # SSR entrypoint (Vercel-compatible fetch handler)
  styles.css            # Tailwind v4 + theme tokens + custom utilities
```

## Key Conventions

- **Server-only code**: Files with `.server.ts` suffix (e.g. `config.server.ts`) are never bundled into the client. `process.env` reads work at any scope (Vercel Node.js runtime).
- **Public config**: `import.meta.env.VITE_*` env vars. No `.env` files committed — define as needed.
- **`no-restricted-imports` blocks `"server-only"`** — use `.server.ts` suffix or `@tanstack/react-start/server-only` instead.
- **`@/*`** path alias maps to `src/` (configured in tsconfig and vite via vite-tsconfig-paths).
- **ESLint**: `@typescript-eslint/no-unused-vars` is OFF — unused vars are fine.
- **Prettier**: double quotes, trailing commas everywhere, 100 char width.
- **Supply chain**: `bunfig.toml` has a 24h `minimumReleaseAge` guard. Exceptions via `minimumReleaseAgeExcludes`.
- **Build artifacts**: `dist`, `.output`, `.vinxi`, `.tanstack`, `.nitro`, `.wrangler/` are all gitignored.

## CSS / Design Tokens

- Tailwind v4 `@theme inline` block in `src/styles.css` defines all design tokens.
- Color system uses **oklch** with green-tinted primary. Legacy aliases: `amber → primary`, `signal → scarcity accent`, `line → border-light`, `dim → muted-foreground`.
- Custom Tailwind utilities: `container-x` (max-width 1200px), `mono-num` (tabular numbers), `eyebrow` (mono uppercase label), `hairline` (fading horizontal rule), `skip-link` (a11y skip-nav).
- Fonts: Geist (display/sans), JetBrains Mono (mono) — loaded from Google Fonts in `__root.tsx`.

## Deployment

- Target: **Vercel** via Nitro's `vercel` preset (`.vercel/output/` build directory).
- Build command: `bun run build` → outputs to `.vercel/output/`.
- Import repo in Vercel dashboard — Vercel auto-detects TanStack Start + Nitro.
- Set environment variables (any `VITE_*` or server-side secrets) in Vercel project settings.
- OG image URL needs replacing before going live (see `src/routes/__root.tsx` for TODO).
